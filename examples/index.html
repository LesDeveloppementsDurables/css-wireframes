Syntax and naming convention

All the framework classes are prefixed with wf- : wf-block, wf-note, etc.
Furthermore, we use the BEM naming convention. In a nutshell, modifiers look like this : element--modifier, and child elements look like this : element__child.
Also, always use modifiers in conjonction to the basic class, like so : class="wf-something wf-something--modifier".
Please note that we use a bit a flexbox, transformations and such, which rules out old outdated browsers.
